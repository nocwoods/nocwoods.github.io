emacs-org with ox-hugo

emacs-org mode with ox-hugo export supported.

ox-hugo

  • ox-hugo is the bridge from emacs-org to hugo’s content. existing .org file with ox-hugo export to .md file and format translate;

parameters:

  • HUGO_BASE_DIR (mandatory) defined the base folder for hugo export;
  • HUGO_SECTION hugo section By default, ox-hugo exports all the markdown files to <HUGO_BASE_DIR>/content/posts/. the content in this folder will be shown with URL/ root path; for other section folders, can be accessed with URL/SECTION-NAME, under <HUGO_BASE_DIR>/content/<SECTION-NAME>

(if there’s another content/post exist, setup mannually, then Hugo server linked to content/post folder instead of content/posts;) with default setting of ox-hugo, we need to access with address like:http://localhost:1313/posts/

reference: hugo-section

  • HUGO_WEIGHT: auto manual set page weight or auto;
  • HUGO_AUTO_SET_LASTMODE: t
  • PROPERTIES: have the oppotunity to export each blog based on org file subtree;
    • EXPORT_FILE_NAME: (mandatory)
    • EXPORT_DATE: (can be exported based on SCHEDULED/CLOSED/DELAYED)
    • EXPORT_HUGO_MENU:
    • EXPORT_HUGO_DRAFT: t (can be exported based on TODO/DONE)
    • EXPORT_STARTUP: content
    • EXPORT_HUGO_CUSTOM_FRONT_MATTER: :mathjax true
  • HUGO: more remmended if hugo theme support “read more” feature;
here's summary or abstract;
#+hugo: more
here's details for next;
  • Categary: can be exported with emacs org tag with @ prefix

insert images and auto copy

  • put images under BASE-DIR/static/images/*.png, update .org file with inserted code

    [[/images/ddr_tRRD.png]]

    NOT-RECOMMENDED for existing org structure;

  • direct link in .org file then ox-hugo will automatically copy image files to /static/ox-hugo/*.png other kinds of variants can be found in reference: image-links

  • examples:

    [[~/Org/images/ddr_tWTR.png]]
➜  static git:(master) ✗ ls ox-hugo
ddr_tVREF_time.png  ddr_tWTR.png
➜  static git:(master) ✗ ls images

equations

  • ox-hugo emacs auto-indent mode need to be careful, otherwise will impact the final mathjax function; please refer to: https://github.com/kaushalmodi/ox-hugo/issues/128
  • update mathjax

    • update file <theme-name>/layout/partial/mathjax.html, to include refered javascript, in order to support equation reference, mathjax.html.

    • meanwhile, create or modify <theme-name>/static/js/mathjax-config.js, to include refered js file: mathjax-config.js

    • insert the line inside <theme-name>/layout/_default/single.html

      {{ partial "mathjax.html" . }}

  • reference equations like: \ref{eq:1}
